# 研报分析 Agent 设计方案

---

## 一、产品定位

**普通投资人的专属投资机构**

目标用户：非金融专业的普通投资人，希望通过基本面分析、商业模式分析等工具，辅助股票投资决策。

### 核心价值主张

| 对比对象 | 他们做什么 | 我们的差异化 |
|---------|-----------|------------|
| 雪球、东方财富 | 行情展示、社区讨论 | 记忆用户画像和投资偏好，制定个性化交易计划并长期跟踪 |
| 聚宽、果仁 | 量化指标筛选 | 覆盖更全面的要素：财报原文、研报观点、社区情绪，细致挖掘公司基本面 |
| 传统投资顾问 | 人工服务，成本高 | AI 驱动，7×24小时，贴近机构研究流程，服务个人投资者 |

**一句话定位：** 让普通投资人用上机构级的研究工具，践行价值投资理念，实现长期稳定收益。

---

## 二、投资理念

以巴菲特、彼得林奇的价值投资为核心框架：

**巴菲特框架**
- 买好公司：有护城河、管理层优秀、自由现金流充裕
- 等好价格：内在价值显著高于当前市价，有安全边际
- 长期持有：买入即准备持有5-10年

**彼得林奇框架**
- 买自己看得懂的生意
- 关注成长性：PEG < 1 是好机会
- 关注被市场忽视的中小盘公司

**核心原则：** 系统只提供客观数据和分析框架，最终决策权在用户手里。系统不给"买入/卖出"信号，只帮用户把事实搞清楚。

## 三、业务流程

```
宏观判断（暂不做）
    ↓
行业判断
  - 行业景气度（PPI、产能利用率、政策方向）
  - 行业竞争格局（龙头市占率、行业集中度）
  - 行业研报
    ↓
个股筛选
  - 量化筛选：市值、PB、PE、ROE、FCF
  - 财务健康度分析：三张报表、自由现金流
  - 商业模式分析：护城河、收入结构、定价权
  - 估值模型：PB历史分位、PE历史分位、DCF（待做）
    ↓
深度研究
  - 年报精读（按章节结构化问答）
  - 研报汇总（机构观点、盈利预测）
  - 社区情绪（雪球评论分析）
    ↓
制定交易计划
  - 确定买入价格区间和仓位
  - 设定持有条件（基本面不变则持有）
  - 设定止损条件（基本面恶化则退出）
    ↓
持仓跟踪
  - 定期财务数据更新
  - 重大公告提醒
  - 基本面变化预警
    ↓
复盘与进化
  - 记录每笔决策的依据
  - 对比实际结果和预期
  - Agent 从反馈中学习，持续提升准确率
```

技术面分析**不纳入**本系统，原因：
1. 技术面预测短期价格，与基本面投资理念冲突
2. 技术面工具已高度成熟，无差异化空间
3. 对目标用户（非专业投资者）而言，技术面反而会增加噪音

## 四、核心功能模块

### 4.1 用户画像（待做）
记录用户的投资风格和偏好，使系统从通用工具变为专属顾问。

```
投资风格：价值 / 成长 / GARP / 不限
风险偏好：保守（低波动）/ 均衡 / 激进（高成长）
关注行业：消费 / 科技 / 医药 / 金融 / 能源 / 不限
投资周期：短期（<1年）/ 中期（1-3年）/ 长期（>3年）
目标收益率：___% / 年
```

### 4.2 个股分析（已有基础）

**数据层**
- 实时行情：Tushare `daily` / `daily_basic`
- 三张财务报表：Tushare `income` / `balancesheet` / `cashflow`
- 年报全文：巨潮资讯（PDF 下载 + 章节化向量入库）
- 券商研报：用户手动上传

**分析层**
- 财务健康度：营收/净利润/FCF 趋势、利润率、负债率
- 自由现金流：FCF = 经营现金流 - 资本支出，FCF/净利润（现金含量）
- 估值分析：PB/PE 历史分位，买入/合理/卖出价格区间
- 知识库问答：基于年报 + 研报的精准问答（章节路由 + 查询扩展）

**待做**
- DCF 估值模型（需用户输入增长率假设）
- ROE 杜邦拆解（净利率 × 资产周转率 × 杠杆倍数）
- 行业横向对比

### 4.3 智能选股（已有基础）

- 量化筛选：市值、PB、PE、ROE 等条件组合
- 综合评分排序（低 PB + 低 PE + 高 ROE）
- 待做：行业过滤、FCF 筛选条件、成长性筛选

### 4.4 投资组合跟踪（待做）

```
持仓记录：
  股票代码 | 买入价 | 持有数量 | 买入日期 | 买入理由

实时收益：
  当前价 | 持仓成本 | 盈亏金额 | 盈亏比例

持仓健康检查（定期触发）：
  基本面是否发生重大变化？
  当前估值是否已进入高估区间？
  是否有重大风险信号（减持、审计意见、业绩预警）？
```

### 4.5 社区情绪分析（待做）

- 爬取雪球评论（指定股票的近期讨论）
- 情感分析：正面 / 中性 / 负面
- 热点话题提取
- 作为辅助判断维度（非决策依据）

### 4.6 投资笔记（已实现）

用户记录自己对公司的看法、分析结论、投资决策依据，形成个人知识资产。

**核心设计**
- 笔记与股票代码绑定，也可以是通用笔记（不绑定特定股票）
- 笔记存入 SQLite（结构化）+ Chroma 向量库（语义检索）
- AI 问答时会自动检索相关笔记，结合用户自己的观点给出回答
- 页面上可以查看、新增、删除历史笔记

**使用场景**
```
"今天看了茅台年报，管理层换届，新CEO更注重渠道改革" → 记录为笔记
"我判断茅台的合理买入价在1400-1500之间，原因是..." → 记录为笔记
问 AI："我之前对茅台有什么判断？" → AI 检索笔记后回答
```

**笔记字段**
```
id, stock_code（可空）, title, content, created_at, updated_at
```

### 4.7 Agent 进化机制（已有基础）

**反馈循环**
- 每条回答下方 👍/👎 反馈
- 差评记录"哪里不对"
- 定期查看 `/api/knowledge/feedback/stats` 找改进方向

**知识库持续完善**
- 用户可手动上传研报
- 季报/年报发布后自动提醒入库
- 章节化拆分 + 查询扩展，持续提升检索准确率

**Agent 变聪明的路径**
```
更好的数据 → 章节化拆分、结构化财务指标、行业数据
反馈循环   → 👍👎记录，定期回顾差评，针对性优化
知识积累   → 高频问题缓存，人工审核后固化为高质量答案
```

## 七、关键设计原则

1. **客观事实优先**：系统的核心价值是把客观数据和事实搞清楚，不替用户做决策
2. **数字必须精确**：所有数学计算由 Python 完成，LLM 只做语言组织
3. **来源可追溯**：每条结论标注数据来源（报告名称 + 章节）
4. **反馈驱动进化**：没有评估体系就没有改进方向，差评是最有价值的数据
5. **长期主义**：系统设计为长期使用，数据和知识随时间不断积累
